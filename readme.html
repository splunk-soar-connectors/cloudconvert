This app integrates with Cloud Convert to convert the files to a specified format.

Port Details

The app uses HTTP/ HTTPS protocol for communicating with the Cloud Convert server. Below are the default ports used by the Splunk SOAR Connector.
SERVICE NAME TRANSPORT PROTOCOL PORT
http tcp 80
https tcp 443

Steps to Configure the Cloud Convert Phantom app's asset

Follow these steps to configure the Cloud Convert Phantom app's asset:

  • Log in to the Cloud Convert platform.
    • Once logged in, select Dashboard by clicking the user's profile logo.
    • In API section, go to Authorization.
    • Click on API Keys and Create new.
    • Now, give any name to your key and give the below mention scopes to it:
      • user.read: View your user data
      • user.write: Update your user data
      • task.read: View your task and job data
      • task.write: Update your task and job data
    • Click on the Create button.
    • Copy the generated API Key.
    • NOTE: The API key generated will be shown only once. So store it safely.
  • Now, Log in to your Phantom instance.
    • Navigate to the Home dropdown and select Apps.
    • Search the Cloud Convert App from the search box.
    • Click on the CONFIGURE NEW ASSET button.
    • Navigate to the Asset Info tab and enter the Asset name and Asset description.
    • Navigate to the Asset Settings.
    • Paste the generated API Key from Cloud Convert UI to its respective configuration parameter.
    • Mention the number of minutes to poll to convert file. Default is 1 minute.
    • Save the asset.
    • Now, test the connectivity of the Phantom server to the Cloud Convert instance by clicking the TEST CONNECTIVITY button.

Explanation of the Asset Configuration Parameters

The asset configuration parameters affect 'test connectivity' and some other actions of the application. The parameters related to test connectivity action is API Key and Number of minutes to poll for converted file.
  • API Key: API Token for asset authorization.
  • Number of minutes to poll for converted file: Number of minutes user wants to take to convert any file. The default time is one minute

Explanation of the Cloud Convert Actions' Parameters

  • Test Connectivity (Action Workflow Details)

    • This action will test the connectivity of the Phantom server to the Cloud Convert instance by making an initial API call using the provided asset configuration parameters.
    • The action validates the provided asset configuration parameters. Based on the API call response, the appropriate success and failure message will be displayed when the action gets executed.

  • Get Valid Filetypes

    In order to retrieve information of valid input-output formats, user can run this action.
    • Action Parameter: Input filetype
      • This is a required parameter. The input filetype supports 200 filetypes. On selecting any one of the input format, the action generates a custom view listing valid output formats for the specified input format.

  • Convert File

    This action lets a user convert file for a valid input-output format.
    • Action Parameter: Vault ID
      • This parameter is the unique id for any file and it is a required parameter. In case if file content of more than one file is same but have different filename then vault id will be same for all those files.
    • Action Parameter: Filetype
      • This parameter is a required parameter and a dropdown box which lists all the output formats in which user wants their file to be converted into.
    • Action Parameter: Input filename
      • This parameter is optional parameter and user need to provide name alongwith extension of the file which has been uploaded in the vault. In case of different vault id of files, user does not need to provide this parameter. If the vault id of more than one file is similar, then user need to provide the input filename of the file uploaded in vault. In case if the vault id is similar and user does not provide the input filename, then output generated file could be any of the similar vault id.
